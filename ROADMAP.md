# 🗺️ Git & GitHub Fundamentals Roadmap

Welcome to the **Git & GitHub Fundamentals** roadmap! This document outlines a complete learning journey, structured from basic concepts to advanced version control techniques. It also serves as an index for resources, cheat sheets, and tutorials that will be maintained in this repository.

---

## 📍 Phase 1: The Basics (Understanding Version Control)
*Learn the core concepts of tracking changes and why version control is essential.*

- [ ] **What is Version Control?** (Git vs. GitHub)
- [ ] **Installation & Setup**
  - Installing Git (Windows, macOS, Linux)
  - First-time configuration (`git config`)
  - Setting up SSH keys / Personal Access Tokens (PAT) for GitHub
- [ ] **Creating Your First Repository**
  - `git init` vs. `git clone`
- [ ] **The Git Workflow (The Three Trees)**
  - Working Directory, Staging Area, Local Repository
  - Adding changes (`git add`) and Committing (`git commit`)
- [ ] **History & Logs**
  - Viewing history (`git log`, `git show`)
  - Viewing differences (`git diff`)

## 📍 Phase 2: Branching & Working Independently
*Learn how to isolate work, manage features, and resolve basic conflicts.*

- [ ] **Branching Basics**
  - Creating and switching branches (`git branch`, `git checkout`, `git switch`)
- [ ] **Merging Changes**
  - Fast-forward vs. 3-way merges (`git merge`)
- [ ] **Handling Merge Conflicts**
  - Identifying, resolving, and completing a conflict resolution

## 📍 Phase 3: Remote Repositories & Collaboration
*Connecting your local code to the internet and collaborating with others.*

- [ ] **Working with Remotes**
  - Adding remotes (`git remote add`)
  - Pushing changes (`git push`)
  - Fetching vs. Pulling (`git fetch`, `git pull`)
- [ ] **The GitHub Flow**
  - Forking repositories
  - Creating Pull Requests (PRs)
  - Code reviews and requesting changes

## 📍 Phase 4: Undoing Mistakes
*Time travel in Git: How to safely undo, revert, and fix blunders without losing work.*

- [ ] **Unstaging & Discarding Changes** (`git restore`)
- [ ] **Modifying Commits** (`git commit --amend`)
- [ ] **Reverting vs. Resetting**
  - Soft, Mixed, and Hard Resets (`git reset`)
  - Safe undoing (`git revert`)
- [ ] **The Git Reflog**
  - Recovering "lost" commits using `git reflog`

## 📍 Phase 5: Advanced Git Techniques
*Power-user commands for maintaining a clean and linear project history.*

- [ ] **Rebasing**
  - `git rebase` vs. `git merge`
  - Interactive Rebasing (`git rebase -i`): squashing, rewording, dropping commits
- [ ] **Stashing**
  - Saving unfinished work temporarily (`git stash`, `git stash pop`)
- [ ] **Cherry-Picking**
  - Applying specific commits to other branches (`git cherry-pick`)
- [ ] **Git Bisect**
  - Debugging by finding which commit introduced a bug

## 📍 Phase 6: Mastering GitHub
*Beyond Git: Utilizing GitHub's ecosystem for project management and automation.*

- [ ] **Project Management**
  - GitHub Issues, Labels, and Milestones
  - GitHub Projects (Kanban boards)
- [ ] **GitHub Pages**
  - Hosting static documentation or workflows directly from a repository
- [ ] **GitHub Actions (CI/CD)**
  - Automating workflows, testing, and deployment
- [ ] **GitHub Security**
  - Dependabot alerts, Secret Scanning, CodeQL
- [ ] **Building an Outstanding GitHub Profile**
  - README formatting, pinned repos, activity graph
